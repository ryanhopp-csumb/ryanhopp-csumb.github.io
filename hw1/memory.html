Memory Management


For memory management, each computer has a virtual address space and a physical address space.
Any process that can access memory only gets to see a set of virtual addresses and no physical addresses.
This means that processes can use more memory than the computer physically has with no issue.
However, this also means that we have to keep some way of translating the virtual addresses to physical addresses.
There are a few different approaches to doing this address translation, such as:

  1. Base and bounds - All virtual addresses for a process are between a base and a bounds.
  2. Segmentation - Processes get different segments of memory for their virtual address space.
  3. Paging - Every process gets some number of memory pages to use, all virtual addresses are on the page(s) given.
